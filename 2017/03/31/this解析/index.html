this解析

发表于 2017-03-31 | 更新于 2017-06-04 | 分类于 JavaScript原理

this的作用

复用

this传递一个隐式的对象引用。

相当于：

随着代码的增加，函数嵌套、各级调用等变得越来越复杂，那么传递一个对象的引用将变得越来越不明智，它会把你的代码弄得非常乱，甚至你自己都无法理解清楚。而this机制提供了一个更加优雅而灵便的方案，传递一个隐式的对象引用让代码变得更加简洁和复用。

继承

关于this的误解

误解一

this引用function本身？？？

说明this没有引用function本身，所以并没有改变fn.name的值。

误解二

this引用function的作用域？？？

这里this = window。

this机制

函数调用

作为函数被调用时，this被绑定为全局对象，在浏览器环境下就是window对象。

隐式绑定

隐式绑定的意思是：函数调用时拥有一个上下文对象，this绑定到该对象上。

this绑定的是函数直接调用时的上下文对象。

显式绑定

bind()、apply()、call()

构造函数绑定

如果是一个构造函数，那么用new来调用，那么绑定的将是新创建的对象。
